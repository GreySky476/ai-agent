# Skill: project-architect-assistant

## 输入：需求卡片

## 方案生成规则

- **基础方案**：满足最小功能，成本最低，适合MVP或个人项目。
- **标准方案**：平衡成本与性能，适合大多数中小团队（推荐）。
- **增强方案**：高可用、可扩展，适合预期快速增长或企业级项目。

### 方案对比表模板（以智能客服为例）

| 维度 | 基础方案 | 标准方案（推荐） | 增强方案 |
|------|----------|------------------|----------|
| 技术栈 | Spring Boot + 内存向量检索 + H2 | Spring Boot + PGVector + Redis + LangChain4j | Spring Cloud + Qdrant集群 + Kafka + 监控 |
| 服务器配置 | 1台 2C4G | 2台 4C8G（应用+缓存） | 4台+ 8C16G + 中间件集群 |
| 成本等级 | 低 | 中 | 高 |
| 开发效率 | ★★★★ | ★★★ | ★★ |
| 可维护性 | ★★★ | ★★★★ | ★★★ |
| 可扩展性 | ★★ | ★★★ | ★★★★★ |
| 适用场景 | 内部测试、低负载 | 生产环境、百级QPS | 千级QPS、多租户 |

### 特殊场景处理

- **高并发电商（场景F）**：基础方案不可用，直接提供“标准方案”和“增强方案”，并说明“若预算极低，建议使用成熟的SaaS电商平台而非自建”。
- **金融/支付（场景I）**：基础方案需要增加安全基线（如强制HTTPS、审计日志、加密），成本等级自动提升一档。

### 输出格式

用户选择方案后，输出完整架构报告，包含：
- 技术栈清单（含版本建议，如Spring Boot 3.2.x LTS）
- 项目结构树
- 安全清单（通用 + 场景专项）
- 常见陷阱（场景相关）
- 快速启动命令

### 成本估算的表述

不使用具体金额，给出相对等级和配置范围。例如：
> 成本等级：中（约需2台4C8G服务器 + Redis，可参考云厂商报价约每月800-1500元，具体视地域和折扣）
